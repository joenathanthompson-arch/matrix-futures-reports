## Signal Thresholds

| Score Range | Signal | PM Code | Description |
|-------------|--------|---------|-------------|
| ≥ +5 | STRONG_BULLISH | 3 | High conviction long bias |
| +3 to +4 | BULLISH | 2 | Lean long |
| +1 to +2 | SLIGHT_BULLISH | 1 | Modest long bias |
| -1 to +1 | NEUTRAL | 0 | No directional edge |
| -2 to -3 | SLIGHT_BEARISH | -1 | Modest short bias |
| -4 to -5 | BEARISH | -2 | Lean short |
| ≤ -6 | STRONG_BEARISH | -3 | High conviction short bias |

## Confidence Scoring

| Confidence | Meaning |
|------------|---------|
| 8-10 | High conviction, fresh data, clear signals |
| 6-7 | Moderate conviction, some uncertainty |
| 4-5 | Low conviction, conflicting signals |
| 1-3 | Very uncertain, recommend reduced size |

## Data Flow

1. **Manus AI** runs daily, fetching macro data from multiple sources
2. **Manus** scores each instrument using weighted factor models
3. **Manus** commits JSON + summary to this repository
4. **Portfolio Manager** reads `latest.json` via GitHub API
5. **PM** sends environment codes to Traders via Global Variables
6. **Traders** combine PM bias with ALMA state + IB conditions
7. **Traders** send strategy commands to Trade Manager
8. **Trade Manager** executes trades

## Critical Rules for Manus

1. **NO HALLUCINATION** - Every data point must come from verified sources
2. **INTEGER MATH ONLY** - All scores are whole numbers, no decimals
3. **DOCUMENT STALENESS** - Mark any data >1 day old in `data_quality`
4. **USE FALLBACKS** - When FRED is stale, use CNBC/TradingView
